Google Analytics Implementation Guide

Complete setup for WordPress/WooCommerce on Cloudflare Tunnel

Introduction & Executive Summary

This document provides a complete, step-by-step technical guide for implementing Google Analytics tracking on a WordPress website with WooCommerce functionality, hosted locally through a Cloudflare Tunnel setup.

The environment consists of a Docker-based WordPress installation running on a local laptop, connected to the internet via Cloudflare's infrastructure using the domain unipi.ai.

The implementation leverages the Google Site Kit plugin to simplify Analytics integration while ensuring proper tracking of WooCommerce activities and customer interactions.

Data Flow Explanation:

  1. A user requests https://unipi.ai
  2. Cloudflare's DNS routes the request to the Cloudflare network
  3. An established Cloudflare Tunnel securely proxies the request to your local Dockerized WordPress instance
  4. WordPress serves the page, which includes WooCommerce elements
  5. The Google Site Kit plugin injects the GA4 tracking code into the page
  6. As the user interacts with the site, tracking events are sent to Google Analytics servers
  7. Data is processed and displayed in both the GA4 and Site Kit dashboards

Prerequisites & Initial Configuration Checklist

Before you begin, ensure your environment is correctly set up:

Docker Environment

  • Docker & Docker-Compose are installed and running
  • WordPress and MySQL containers are operational
  • Volumes are configured for data persistence

Cloudflare Tunnel

  • The Cloudflare CLI (cloudflared) is installed on your local machine
  • You are logged in to your Cloudflare account via cloudflared tunnel login
  • A tunnel is created: cloudflared tunnel create unipi-tunnel
  • A configuration file is set up in ~/.cloudflared/
  • The tunnel is routed to your DNS
  • The tunnel is running (or configured as a service)

WordPress & DNS

  • WordPress is fully installed and accessible via https://unipi.ai
  • The SSL certificate provided by Cloudflare is valid
  • The domain unipi.ai is managed by Cloudflare with proxying enabled

Step-by-Step Implementation Guide

1Create a Google Analytics 4 Property

  1. Navigate to Google Analytics
  2. Click "Admin" (gear icon in the lower left)
  3. In the "Account" column, ensure your desired account is selected
  4. In the "Property" column, click "Create Property"
  5. Enter Property Details:
    • Property name: unipi.ai WooCommerce Demo
    • Reporting time zone: (Select your time zone)
    • Currency: (Select your currency)
  6. Click "Next" and provide information about your business
  7. Click "Create"
  8. Set up a Data Stream: After creation, select "Web"
    • Website URL: https://unipi.ai
    • Stream name: unipi.ai Web Stream
  9. Enable Enhanced Measurement
  10. Record Your Measurement ID: Note the Measurement ID (e.g., G-XXXXXXXXXX)

2Install and Configure Google Site Kit Plugin

  1. In your WordPress admin dashboard, navigate to Plugins > Add New
  2. Search for "Site Kit by Google"
  3. Click "Install Now" and then "Activate"
  4. Click "Start Setup" on the welcome screen
  5. Authenticate with your Google account
  6. Grant permissions to Site Kit
  7. Verify site ownership (handled automatically)
  8. Select your GA4 property from the list
  9. Complete the configuration

3Configure Enhanced E-commerce Tracking in WooCommerce

  1. Install the "WooCommerce Google Analytics Integration" plugin
  2. Navigate to WooCommerce > Settings > Integrations > Google Analytics
  3. Paste your GA4 Measurement ID into the "Google Analytics ID" field
  4. Enable these critical features:
    • Enable Standard Tracking
    • Enable Enhanced eCommerce Tracking
    • Add to Cart Event
    • Remove from Cart Event
    • Enable Google Analytics Product Attribution
  5. Save your changes

Pro Tip: Test each step as you go to identify any issues early in the process.

Verification and Testing

Ensure everything is working correctly before relying on the data:

Real-Time Report Test

  1. In your Google Analytics dashboard, go to Reports > Real-Time
  2. Open your site in an Incognito/Private browser window
  3. You should see your active user appear in the Real-Time report within 30 seconds

E-commerce Action Test

  1. In the same Incognito window, perform test actions:
    • View a product
    • Add it to the cart
    • Begin the checkout process
  2. In Real-Time > Conversions, you should see events fire

Tag Assistant Check

  1. Install the "Google Tag Assistant" Chrome extension
  2. Visit your site
  3. The extension will show which tags are firing and report any errors

Troubleshooting Common Issues

Issue Symptom Solution
No Data in Reports Real-Time works, but other reports are empty Wait 24-48 hours for reports to populate
SSL/Certificate Error Browser warnings or mixed content errors Ensure WordPress URL settings use https://unipi.ai
Tunnel Connection Drop Site becomes inaccessible Restart the tunnel or configure it as a service
E-commerce Events Missing Pageviews work but no purchase events Re-check WooCommerce integration plugin settings

Warning: Always test your configuration thoroughly before relying on analytics data for business decisions.
